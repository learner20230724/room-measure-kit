# 快速上手 — Room Measure Kit

2 分钟内完成房间估算。无需账号、无需安装 — 打开应用，输入尺寸即可。

**在线访问：** https://learner20230724.github.io/room-measure-kit/

---

## 第一步 — 选择单位

点击工具栏右上角的 **m / ft** 切换按钮，在公制和英制之间切换。

| 模式 | 长/宽/高 | 损耗率 | 涂布率 |
|------|---------|--------|--------|
| 公制 | 米（m） | % | m²/L |
| 英制 | 英尺（ft） | % | ft²/gal |

单位偏好会保存到浏览器 localStorage，关闭后重新打开仍然有效。

---

## 第二步 — 使用预设（可选）

工具栏有三组**预设按钮**，一键填入典型房间尺寸：

- **小卧室** — 4m × 3m × 2.5m（欧盟标准）
- **客厅** — 6m × 4m × 2.8m（宽敞型）
- **工作室** — 8ft × 10ft × 9ft（美国标准）

点击任意预设，所有输入框自动填入对应数值，然后可以手动修改为实际房间尺寸。

---

## 第三步 — 输入房间尺寸

逐项填写输入框：

| 字段 | 含义 |
|------|------|
| **长度（Length）** | 最长墙面，地面水平长度 |
| **宽度（Width）** | 最短墙面，地面水平宽度 |
| **高度（Height）** | 墙面高度（地面到天花板） |
| **地板损耗 %** | 切割损耗余量（矩形房间通常 10–15%） |
| **涂料覆盖率** | 涂料能覆盖的面积（查看油漆桶标签；参考：10 m²/L 或 400 ft²/gal） |

结果随输入实时更新，无需点击"计算"按钮。

---

## 第四步 — 查看结果

同时显示五项计算结果：

| 结果 | 计算公式 |
|------|---------|
| **地板面积（Floor area）** | 长度 × 宽度 |
| **周长（Perimeter）** | 2 × (长度 + 宽度) |
| **墙面面积（Wall area）** | 周长 × 高度 |
| **地板需求量（Flooring needed）** | 地板面积 × (1 + 损耗率) |
| **涂料需求量（Paint needed）** | 墙面面积 ÷ 涂料覆盖率 |

> **注意：** 墙面面积按四面完整墙体计算，未扣除门窗。它是规划估算值，不是施工报价。

## 第五步 — 分享链接

点击工具栏的 **分享链接（Share link）** 按钮，当前输入参数会编码到 URL 中。复制链接后可以粘贴到聊天、邮件或笔记里。任何人打开该链接，都会看到相同的房间配置。

链接格式：`https://learner20230724.github.io/room-measure-kit/?state=<编码>`

## 第六步 — 导出结果

在结果面板顶部，有两个导出按钮：

- **PNG** — 以高分辨率（2×）截取结果面板图片，背景为深色主题。
- **PDF** — 生成结果面板的 PDF 文档，方向（横向/纵向）根据布局自动判断。

## 常见使用场景

### 购买地板材料
输入房间尺寸，设置地板损耗率 10–15%，用**地板需求量**作为订购数量。向上取整到整包。

### 规划油漆用量
输入尺寸，根据油漆桶标签设置涂料覆盖率（如 10 m²/L），用**涂料需求量**估算升数。大多数房间需要涂两遍 — 将结果乘以 2。

### 房产 Listing 描述
点击**复制摘要（Copy summary）**按钮，生成一行结果摘要，可以直接粘贴到房源描述或聊天中。

## 常见问题

**问：可以计算非矩形房间吗？**
暂不支持。L 型和圆形房间在路线图中（M6），目前计划中。暂时可以将 L 型房间拆分为两个矩形，分别计算后相加。

**问：英制结果看起来不对，应该用多大的覆盖率？**
油漆覆盖率因品牌而异。美国乳胶漆通常标注为约 400 ft²/gal。请查看油漆桶标签上的覆盖率说明。

**问：应用支持离线使用吗？**
支持。v0.1.1 已加入 PWA 支持，首次加载后应用外壳会被缓存，之后的访问无需网络。

**问：我的数据存在哪里？**
所有数据都存储在浏览器 localStorage（`room-measure-kit-history`），不会发送到任何服务器。

cd room-measure-kit

## 技术栈

Room Measure Kit 基于 React 19 + TypeScript + Vite 构建。无后端、无外部依赖（React 本身除外）。整体以静态站点形式交付，可部署到 GitHub Pages、Cloudflare Pages、Netlify 或任何静态托管环境。
